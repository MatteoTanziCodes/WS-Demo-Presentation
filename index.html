WHAT IS A BACKGROUND AGENT

The answer: agents that review
before any human has to. ⬡ ✳ ✦ ○

A copilot needs your analyst's attention. A background agent needs neither. It runs in its own execution environment in the cloud: full document toolchain, policy retrieval suite, everything. Completely decoupled from your team's attention.

Trigger it from a Slack message, a form submission, a webhook, a scheduled job. Walk away. Review the structured output when it's done. It's an asynchronous task: delegate, walk away, decide later.

01
Establish background agent primitives

Autonomous agents need infrastructure that doesn't exist in a shared drive. The building blocks below are what separate a demo from a deployment — structured document ingestion, policy retrieval, sandboxed execution, audit trail generation. Each one unlocks the next.

PATTERN 1

Agent has full document environment

The agent has a full document processing environment. A container running with the policy corpus, test cases, regulatory databases, and internal API access. This is the closest to how a human analyst works. Every enterprise compliance agent worth deploying chose this pattern.

GOVERNANCE

Enforced at runtime, not by prompt

Agents are actors in your system. They need the same controls as human contributors — identity, permissions, audit trails. Governance enforced by a system prompt ("please don't approve edge cases") is a suggestion. Governance enforced at the execution layer — scoped credentials, deterministic command blocking — is actual governance.

WHERE AI MUST STOP

One decision must
remain human — and why.

PASS takes on real cognitive responsibility: parsing, retrieval, initial analysis, exception flagging, audit documentation. But the architecture is explicit about where AI authority terminates.

Critical Human Decision

Final regulatory approval on novel product types

PASS can review a submission against known policy. It cannot determine whether a genuinely novel financial product is appropriate for Wealthsimple's regulatory posture, client base, and risk appetite. That judgment requires institutional knowledge, stakeholder accountability, and regulatory relationship context that no agent should hold. The analyst signs. The agent doesn't.
